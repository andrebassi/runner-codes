## Schema

The job response is a JSON object returned from the guest runner to the host agent after code execution.

```json
{
  "trace_id": "tr-1764388648079963068",
  "stdout": "Hello, World!\n",
  "stderr": "",
  "exit_code": 0,
  "error": ""
}
```

## Fields

<ResponseField name="trace_id" type="string" required>
  The same trace_id from the request, for correlation.

  **Example:** `"tr-1764388648079963068"`
</ResponseField>

<ResponseField name="stdout" type="string" required>
  Standard output from the executed code.

  **Notes:**
  - Contains all output written to stdout
  - Newlines are preserved as `\n`
  - May be empty if no output was produced
  - UTF-8 encoded

  **Example:** `"Hello, World!\n"`
</ResponseField>

<ResponseField name="stderr" type="string" required>
  Standard error output from the executed code.

  **Notes:**
  - Contains compiler/interpreter errors and warnings
  - Contains runtime errors
  - Contains timeout messages
  - May be empty on successful execution

  **Example:** `"SyntaxError: invalid syntax\n"`
</ResponseField>

<ResponseField name="exit_code" type="integer" required>
  The exit code from the executed process.

  | Code | Meaning |
  |------|---------|
  | 0 | Success |
  | 1 | General error / runtime error |
  | 2 | Misuse of shell command |
  | 124 | Timeout exceeded |
  | 127 | Language not supported |
  | 128+N | Killed by signal N |

  **Example:** `0`
</ResponseField>

<ResponseField name="error" type="string">
  High-level error description (optional).

  **Notes:**
  - Set for system-level errors (not code errors)
  - Examples: "unsupported language", "compilation failed"
  - Empty for successful execution and runtime errors

  **Example:** `"compilation failed"`
</ResponseField>

## Error Handling

### Error Categories

<AccordionGroup>
  <Accordion title="Syntax Errors">
    **Characteristics:**
    - `exit_code`: 1 (or language-specific)
    - `stderr`: Contains parser/compiler error message
    - `error`: Empty (handled by language runtime)

    **Handling:**
    ```python
    if result['exit_code'] != 0 and 'SyntaxError' in result['stderr']:
        # Handle syntax error
        show_user_error("Syntax error in code")
    ```
  </Accordion>

  <Accordion title="Runtime Errors">
    **Characteristics:**
    - `exit_code`: 1 (typically)
    - `stderr`: Contains stack trace
    - `error`: Empty

    **Common types:**
    - ZeroDivisionError
    - TypeError
    - NameError
    - IndexError
    - KeyError

    **Handling:**
    ```python
    if result['exit_code'] != 0 and 'Traceback' in result['stderr']:
        # Handle runtime error
        parse_traceback(result['stderr'])
    ```
  </Accordion>

  <Accordion title="Timeout Errors">
    **Characteristics:**
    - `exit_code`: 124
    - `stderr`: "Execution timed out"
    - `error`: Empty

    **Handling:**
    ```python
    if result['exit_code'] == 124:
        # Handle timeout
        show_user_error("Code took too long to execute")
    ```
  </Accordion>

  <Accordion title="System Errors">
    **Characteristics:**
    - `exit_code`: 127 (language not found) or other
    - `stderr`: System error message
    - `error`: Non-empty description

    **Handling:**
    ```python
    if result['error']:
        # Handle system-level error
        log_error(f"System error: {result['error']}")
    ```
  </Accordion>
</AccordionGroup>

### Best Practices

```python
def handle_execution_result(result):
    """Handle execution result with proper error categorization."""

    # Check for success
    if result['exit_code'] == 0:
        return {
            'status': 'success',
            'output': result['stdout']
        }

    # Check for timeout
    if result['exit_code'] == 124:
        return {
            'status': 'timeout',
            'message': 'Execution exceeded time limit'
        }

    # Check for unsupported language
    if result['exit_code'] == 127:
        return {
            'status': 'error',
            'message': result['error'] or 'Unsupported language'
        }

    # Check for compilation failure
    if result['error'] == 'compilation failed':
        return {
            'status': 'compilation_error',
            'message': result['stderr']
        }

    # Runtime error
    return {
        'status': 'runtime_error',
        'message': result['stderr']
    }
```

## Wire Format

The response uses the same length-prefix framing as the request:

```
+------------------+----------------------+
| Length (4 bytes) | JSON Response        |
| Big-Endian       | (UTF-8 encoded)      |
+------------------+----------------------+
```

### Go Implementation

```go
func recvResult(conn net.Conn) (*Result, error) {
    // Read length prefix
    lengthBuf := make([]byte, 4)
    if _, err := io.ReadFull(conn, lengthBuf); err != nil {
        return nil, err
    }
    length := binary.BigEndian.Uint32(lengthBuf)

    // Read payload
    payload := make([]byte, length)
    if _, err := io.ReadFull(conn, payload); err != nil {
        return nil, err
    }

    // Parse JSON
    var result Result
    if err := json.Unmarshal(payload, &result); err != nil {
        return nil, err
    }

    return &result, nil
}
```

### Python Implementation

```python
import json
import struct

def recv_result(sock):
    # Read length prefix
    length_bytes = sock.recv(4)
    if len(length_bytes) < 4:
        raise Exception("Connection closed")

    length = struct.unpack('>I', length_bytes)[0]

    # Read payload
    payload = b''
    while len(payload) < length:
        chunk = sock.recv(length - len(payload))
        if not chunk:
            raise Exception("Connection closed")
        payload += chunk

    # Parse JSON
    return json.loads(payload.decode('utf-8'))
```
