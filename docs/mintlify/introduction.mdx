## What is LLM-Firecracker?

LLM-Firecracker is a **secure execution environment** for running code snippets generated by Large Language Models (LLMs) inside [Firecracker microVMs](https://firecracker-microvm.github.io/). It provides strong isolation, fast boot times, and support for multiple programming languages.

## Why Firecracker?

Firecracker is a Virtual Machine Monitor (VMM) developed by AWS that provides:

- **Strong isolation**: Full VM-level isolation, not just container namespaces
- **Minimal footprint**: ~5MB memory overhead per microVM
- **Fast startup**: Boots a VM in ~125ms
- **Security-first design**: Built with a minimal device model and seccomp filters

## Architecture Overview

<Frame>
  <img src="/images/architecture-overview.svg" alt="LLM-Firecracker Architecture" />
</Frame>

The system is built around a unified CLI `infra.operator`:

| Component | Description |
|-----------|-------------|
| **infra.operator host** | Controls Firecracker via API socket, starts VMs, sends jobs via vsock |
| **infra.operator guest** | Runs inside microVM, listens on vsock:5000, executes code |
| **Rootfs Images** | Pre-built filesystem images with language runtimes |

## Key Features

<AccordionGroup>
  <Accordion title="Secure Code Execution">
    Code runs inside isolated Firecracker microVMs with:
    - No network access by default
    - Read-write rootfs fully contained within VM
    - VM destroyed after each execution
    - Optional jailer for additional hardening
  </Accordion>

  <Accordion title="Per-Language Optimization">
    Each supported language has its own optimized rootfs image:
    - **Python**: 600MB with Python 3.10 and pip
    - **Node.js**: 700MB with Node.js 20 LTS and npm
    - **Go**: 700MB with Go 1.22.3
    - **Rust**: 2GB with rustc and cargo
    - **Bash**: 512MB minimal image
  </Accordion>

  <Accordion title="S3-Backed Storage">
    Rootfs images are stored in S3 for:
    - Fast deployment to new EC2 instances
    - Version control and rollback
    - Reduced instance storage requirements
    - Easy distribution across regions
  </Accordion>

  <Accordion title="Vsock Communication">
    Host-guest communication via virtio-vsock:
    - No network stack overhead
    - Direct VM-to-host communication
    - 4-byte length-prefixed JSON protocol
    - Low latency job submission
  </Accordion>
</AccordionGroup>

## Quick Example

Here's a simple example of executing Python code in a Firecracker microVM:

<CodeGroup>
```bash CLI
# Execute Python code via CLI
infra.operator run --lang python --code "print('Hello from Firecracker!')"
```

```bash HTTP API
# Start API server
infra.operator api --port 8080

# Execute via HTTP
curl -X POST http://localhost:8080/api/v1/execute \
  -H "Content-Type: application/json" \
  -d '{"language": "python", "code": "print(\"Hello from Firecracker!\")"}'
```
</CodeGroup>

**Output:**
```json
{
  "stdout": "Hello from Firecracker!\n",
  "stderr": "",
  "exit_code": 0,
  "load_time": "209ms",
  "exec_time": "45ms"
}
```
